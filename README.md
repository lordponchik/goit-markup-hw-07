## Eligibility criteria for a mentor

### Project

**`¬´A1¬ª`** The project's HTML code was refactored using the BEM methodology.

**`¬´A2¬ª`** The CSS code of the project has been refactored using the `SASS` preprocessor.

**`¬´A3¬ª`** The `sass` folder is created in the root of the project, in which all the preprocessor style files are located.

**`¬´A4¬ª`** The `sass` folder contains the `main.scss` file - the main file into which all SASS fragments (partials, `_name.scss` files) are imported.

**`¬´A5¬ª`** The layout color palette and font sets are placed in variables in the `variables.scss` file, which is located in the `sass/utils` folder. You can use CSS or SASS variables (optional).

**`¬´A6¬ª`** For each component, a separate styles fragment file has been created in the `sass/components` folder. For example `_page-header.scss`, `_logo.scss` etc.

**`¬´A7¬ª`** The `index.html` and `portfolio.html` files include the minified style file `main.min.css` from the `css` folder.

### Markup

**`¬´B1¬ª`** Correct naming of block classes according to BEM methodology.

**`¬´B2¬ª`** Correct naming of element classes according to BEM methodology.

**`¬´B3¬ª`** Correct naming of modifier classes according to BEM methodology.

**`¬´B4¬ª`** Correct naming of impurity classes according to BEM methodology.

**`¬´B5¬ª`** Class names according to the BEM methodology are clear and descriptive, in English.

## Formalization

**`¬´C1¬ª`** Selector nesting is used.

**`¬´C2¬ª`** The maximum nesting of selectors is 2 levels.

**`¬´C3¬ª`** The concatenation operator (`&`) is used to describe pseudo-classes and pseudo-elements.
